# Feature Definition Of Done

A feature is done only when all checklist points are complete.

## Backend
- migration added (if schema changes)
- validation rules added
- business logic implemented in module service/use-case
- endpoint implemented in `v1`
- automated tests added

## API Contract
- request/response documented in `docs/api/`
- error codes documented
- breaking changes noted

## Mobile
- feature module created/updated (`data/domain/presentation`)
- loading, empty, error states implemented
- tests added (unit/widget where relevant)

## Release
- changelog entry added
- manual QA checklist completed
